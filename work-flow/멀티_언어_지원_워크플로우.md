# 다국어 지원 구현 및 확장 가이드

이 워크플로우는 Java/SQL 외의 다양한 언어와 프레임워크(Go, Swift, Kotlin, C, Python, Node.js)에 대한 분석 기능을 구현하고 확장하는 절차를 설명합니다.

## 1. 아키텍처 개요

`ProjectParser` 인터페이스를 구현하여 각 언어별 파서를 추가합니다. 각 파서는 다음 역할을 수행합니다:
1.  **Supports**: 해당 프로젝트가 이 파서를 지원하는지 확인 (파일 확장자, 특정 파일 존재 여부 등).
2.  **DetectInfo**: 프로젝트의 메타데이터(언어, 프레임워크, 파일 수 등) 감지.
3.  **Parse**: 프로젝트 파일을 읽어 DB 스키마(`TableMetadata`) 추출.

## 2. 지원되는 언어 및 감지 방식

| 언어 | 프레임워크 | 감지 파일 | 파싱 방식 |
| :--- | :--- | :--- | :--- |
| **Java** | JPA (Spring Data) | `.java` (@Entity) | JavaParser AST |
| **SQL** | DDL Scripts | `.sql` (CREATE TABLE) | JSQLParser / Regex |
| **Python** | Django | `models.py` | Regex (`class X(models.Model)`) |
| **Node.js** | Prisma | `schema.prisma` | Parsing `model block` |
| **Go** | GORM | `.go` (struct) | Regex (`type X struct`) |
| **Swift** | CoreData | `.xcdatamodeld` | XML Parsing |
| **Kotlin** | JPA / Room | `.kt` | Regex (`@Entity class`) |
| **C/C++** | Embedded SQL | `.c`, `.cpp`, `.h` | Regex (`CREATE TABLE` String) |

## 3. 구현 단계

### 3.1 파서 클래스 생성
`com.itdg.analyzer.service.parser` 패키지에 각 언어별 파서 클래스를 생성하고 `@Component`로 등록합니다.

```java
@Component
public class MyLanguageParser implements ProjectParser {
    // implements supports, detectInfo, parse
}
```

### 3.2 의존성 관리
복잡한 파싱(AST)이 필요한 경우 `build.gradle`에 의존성을 추가합니다. (예: JavaParser).
단, 경량화 및 종속성 최소화를 위해 Regex나 내장 XML Parser를 우선 사용합니다.

### 3.3 프론트엔드 아이콘 추가
`SchemaReviewStep.jsx`에서 감지된 언어 코드에 맞춰 아이콘(이모지 등)을 매핑합니다.

## 4. 확장 방법 (새로운 언어 추가)
1.  새로운 파서 클래스 (`XXProjectParser`) 생성.
2.  `supports()` 메서드에 감지 로직 구현.
3.  `parse()` 메서드에 스키마 추출 로직 구현.
4.  Spring Context가 자동으로 감지하여 분석 체인에 추가함.
