# 리포지토리 분석 및 대화형 데이터 생성 기능 구현

사용자의 요청에 따라 **GitHub 리포지토리** 및 **로컬 파일**을 분석하여 테스트 데이터를 생성하는 기능을 구현했습니다. 또한 **분석 결과 검토 페이지**와 **010 휴대폰 번호 생성 규칙**을 추가했습니다.

## 주요 변경 사항

### 1. 백엔드 분석 엔진 (`itdg-analyzer`)
- **Git Cloning & Local Upload**: `JGit`을 사용하여 원격 리포지토리를 복제하거나 업로드된 zip 파일을 분석합니다.
- **Java JPA Parser**: `JavaParser`를 도입하여 소스 코드 내 `@Entity` 클래스를 찾아 테이블 스키마를 추출합니다.
- **SQL DDL Parser**: `JSQLParser`를 사용하여 `.sql` 파일의 `CREATE TABLE` 구문을 분석합니다.
- **언어 자동 감지**: 분석된 파일 확장자를 기반으로 프로젝트 언어(Java 또는 SQL)를 식별합니다.

### 2. 프론트엔드 UI (`itdg-frontend`)
- **마법사(Wizard) UI**: `Source Selection` -> `Schema Review` -> `Generation Result` 3단계 흐름으로 개선했습니다.
- **소스 선택**: DB 접속 외에 GitHub URL 입력 및 파일 업로드 탭을 추가했습니다.
- **스키마 검토**: 분석된 테이블 목록과 감지된 언어를 배지 형태로 보여주고, 데이터를 생성할 테이블을 선택할 수 있습니다.

### 3. 오케스트레이션 및 생성 (`itdg-orchestrator`, `itdg-generator`)
- **Metadata 기반 생성**: DB 연결 없이 분석된 메타데이터만으로 데이터를 생성하도록 엔드포인트를 확장했습니다.
- **휴대폰 번호 규칙**: 휴대폰 관련 컬럼(phone, mobile 등)에 대해 `010-XXXX-XXXX` 형식의 난수를 생성하도록 개선했습니다.
